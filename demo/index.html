Are you a starveling?

You should try starveling if you can't swallow a bulky CSS framework for your next website or web-based project

Starveling is a superset derivative of skeleton.css minimally adding a little meat to skeleton's bones. If you are familiar with skeleton then you already are a starveling and should find this CSS boilerplate to your taste.

Typography

Starveling's typography is entirely sized using root ems rem, so font-sizes and spacial relationships can be responsively sized based on a single <html> font-size property. Out of the box, Starveling doesn't change the <html> font-size, but it's there in case you need it for your project. All measurements are still base 10 though so, an <h1> with 5.0remfont-size just means 50px.

Starveling's typograpy base is Titillium Web set at 2rem over a 1.6 line height.

Titillium Web, served by Google, is set at 2rem (20px) over a 1.6 line height (32px). Other type basics like anchors, strong and underline are included and styled specifically for starveling. To ensure the font resource file sizes are wizened an atrophied, emphasis has been made as an equivalent to strong in our CSS. This can easily be overidden if you choose to feed starveling with some jucier font resources.

Headings create a family of distinct sizes each with specific letter-spacing, line-height, and margins. Blockquote's following a sibling heading element are also beautifully styled to suit their place in the document.

Starveling also works well with the Raleway font.

Media Queries

Starveling uses media queries to serve its scalable grid, but also has a list of queries for convenience of styling your site across devices. The queries are mobile-first, meaning they target min-width. Mobile-first queries are how Starveling's grid is built and is the preferrable method of organizing CSS. It means all styles outside of a query apply to all devices, then larger devices are targeted for enhancement. This prevents small devices from having to parse tons of unused CSS. The sizes for the queries are:

  • Desktop HD: 1200px
  • Desktop: 1000px
  • Tablet: 750px
  • Phablet: 550px
  • Mobile: 400px

Utilities

Starveling has a number of small utility classes that act as easy-to-use helpers. Sometimes it's better to use a utility class than create a whole new class just to float an element.


    /* Utility Classes */

    /* Make element full width */
    .u-full-width {
      width: 100%;
      box-sizing: border-box;
    }

    /* Make sure elements don't run outside containers (great for images in columns) */
    .u-max-full-width {
      max-width: 100%;
      box-sizing: border-box;
    }

    /* Float either direction */
    .u-pull-right {
      float: right;
    }
    .u-pull-left {
      float: left;
    }

    /* Clear a float */
    .u-cf {
      content: "";
      display: table;
      clear: both;
    }

    /* Centered text */
    .u-center-text {
      text-align: center;
    }

    /* Additional padding at bottom */
    .u-pad-below {
      padding-bottom: 2rem;
    }

    /* Apply margin to all direct children */
    .u-well > * {
      margin: 0.5rem;
    }
